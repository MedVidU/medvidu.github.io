MedVidU Challenge Guide

Step-by-step instructions for participating in the MedVidU Challenge.

The MedVidU Challenge evaluates models on a strictly held-out test set across multiple medical-video tasks. Submissions are made by sending a Docker image of your model, which we then run internally on the held-out data. Follow the steps below.

1

Download data and train your model

Download the MedVidU validation data from Hugging Face. You are free to train and validate on this data together with any other open-source or internal data of your choice.

While developing, you are welcome to submit predictions to our public community leaderboard for validation feedback.

Model size constraint Important
Submitted models must not exceed 7B parameters.

2

Download the evaluation Docker image Coming soon

We will release a base Docker image with the evaluation harness and required entry-point interface. Build your submission on top of this image so that it is compatible with our internal evaluation pipeline.

Target release: May 20, 2026.

3

Send the signed data-use agreement

Before submitting, you must send a signed copy of the data-use and challenge participation agreement to the organizers. The agreement template will be provided alongside the Docker image release.

Email the signed PDF to the challenge contact address (see the Contact section on the main page).

4

Submit your Docker image

Package your trained model inside the provided Docker image and submit it to the organizers by email. We will run your container on the held-out test set and update the official leaderboard.
